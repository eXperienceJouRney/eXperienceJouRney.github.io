robot

属性

  • name = "名称"

    指定机器人模型的名称

link

用于描述机器人某个部件(也即刚体部分)的外观和物理属性，机器人底座、轮子、激光雷达、摄像头等每一个部件都对应一个 link, 在 link 标签内，可以设计该部件的形状、尺寸、颜色、惯性矩阵、碰撞参数等一系列属性.

属性

  • name = "名称"

    指定连杆的名称

origin

属性

  • xyz="x y z"

    定义该 link 的位置，相对于其父 link 或全局坐标系

  • rpy="roll pitch yaw"

    定义该 link 的旋转，使用滚动（roll）、俯仰（pitch）和偏航（yaw）角

inertial

  • 定义该 link 的惯性属性，用于物理模拟。

标签

  • <mass value = "mass_value">

    定义该 link 的质量

  • <inertia ixx="ixx" ixy="ixy" ixz="ixz" iyy="iyy" iyz="iyz" izz="izz">

    定义该 link 的惯性矩，包括六个分量：ixx, ixy, ixz, iyy, iyz, izz。

visual

  • 定义该 link 的视觉属性，用于渲染。
  • <origin>：定义视觉几何形状的位置和旋转。
  • <geometry>：定义视觉几何形状，可以是 box（盒子）、cylinder（圆柱）、sphere（球体）或 mesh（网格）。
  • <material>：定义视觉材质，可以引用预定义的材质。

geometry

标签

  • <box size="x y z">

    属性:size=长(x) 宽(y) 高(z)

  • <cylinder radius="radius" length="length">

    属性:radius=半径 length=高度

  • <sphere radius="radius">

    属性:radius=半径

  • <mesh filename="filename" scale="x y z">

    属性:filename=文件路径 scale=范围

origin

属性

  • origin xyz="x y z" rpy="roll pitch yaw"

    属性1: xyz=偏移 偏移 偏移

    属性2: rpy=翻滚 俯仰 偏航 (单位是弧度)

metrial

属性

  • name = "名称"

    指定材料的名称

标签

  • <color rgba = "r g b a">

    属性: rgba=红绿蓝权重值与透明度 (每个权重值以及透明度取值[0,1])

collision

  • 定义该 link 的碰撞属性，用于物理模拟中的碰撞检测。
  • <origin>：定义碰撞几何形状的位置和旋转。
  • <geometry>：定义碰撞几何形状，可以是 box（盒子）、cylinder（圆柱）、sphere（球体）或 mesh（网格）。

joint

在 URDF（Unified Robot Description Format）中，joint 元素是机器人结构的连接单元，它定义两个 link 元素之间的连接方式和运动学/动力学属性。通过 joint 可以描述机器人关节的类型、运动范围、惯性参数等，是实现机器人运动模拟和控制的关键。

属性

  • name = "名称"

    指定关节的名称

  • type="形式"

    这是 joint 元素最关键的属性，不同类型需要不同的子元素。

类型 说明 适用子元素 适用场景
fixed 固定关节（不允许运动） 无额外元素 用于连接静态部件（如机器人底座、外壳等）
revolute 旋转关节（绕一个轴旋转） <axis>, <limit> 最常用的旋转关节（如机械臂关节、轮轴）
continuous 连续旋转关节（无限旋转，类似 revolute 但无限制） <axis> 用于需要无限旋转的结构（如电机驱动轴、雷达平台）
prismatic 滑动关节（沿一个轴滑动） <axis>, <limit> 用于线性滑动结构（如伸缩臂、升降机构、电梯导轨）
planar 平面关节（在 XY 平面上自由平移和旋转） <axis>（通常不使用） 用于平面移动平台或二维滑行机构（如移动滑台）
floating 浮动关节（6 DOF，完全自由） 无额外元素 用于基座自由运动（如无人机、潜艇或自由浮动基座）
ball 球关节（3 DOF，类似人体肩膀或髋关节） <axis>（通常不使用） 用于需要球面旋转的连接（如仿人肩关节、球窝接头）

标签

  • <parent>base_link</parent>

    定义关节的父连接（必须有）

  • <child>arm_link</child>

    定义关节的子连接（必须有）

  • <origin xyz="x y z" rpy="roll pitch yaw" />

    • xyz：关节相对于父链接的平移向量（米）
    • rpy：关节相对于父链接的旋转欧拉角（弧度）

  • <axis xyz="x y z"

    表示关节的运动轴（单位向量）

  • <axis xyz="x y z"

    表示关节的运动轴（单位向量）

  • <limit lower="l" upper="u" effort="e" velocity="v" />

    • 描述：定义关节的运动范围、速度和力矩限制。主要用于 revolute 和 prismatic 关节。
    • 参数：
    • lower：最小角度（revolute）或最小滑动距离（prismatic）。
    • upper：最大角度或最大滑动距离。
    • velocity：最大速度（弧度/秒或米/秒）。
    • effort：最大力矩（牛顿米）或推力（牛顿）。

  • <dynamics damping="d" friction="f" />

    • 描述：定义关节的阻尼和摩擦系数，用于物理模拟（如 Gazebo）。
    • 参数：
    • damping：阻尼系数。
    • friction：摩擦系数。

  • <safety k_gain="100" k_velocity="1.0" k_position="1.0" />

    • 描述：定义关节的安全控制参数，用于控制器（如 ros_control）防止过载。
    • 参数：
    • k_gain：安全增益。
    • k_velocity：速度安全系数。
    • k_position：位置安全系数。

base_footprint优化

前面实现的机器人模型是半沉到地下的，因为默认情况下底盘的中心点位于地图原点上。
可以使用的优化策略：将初始 link 设置为一个尺寸极小的 link(比如半径为 0.001m 的球体，或边长为 0.001m 的立方体)，然后再在初始 link 上添加底盘等刚体，这样实现，虽然仍然存在初始link半沉的现象，但是基本可以忽略了。
这个初始 link 一般称之为 base_footprint。
